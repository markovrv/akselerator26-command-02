Ты выступаешь в роли Fullstack-архитектора и тимлида. Тебе необходимо составить детальный пошаговый план разработки веб-платформы «Вперёд по маршрутам промышленности» согласно предоставленному Техническому заданию (ТЗ-02).

**Технологический стек:**
- **Frontend:** Vue 3 (Composition API), TypeScript, Pinia (state management), Vue Router.
- **Backend:** Node.js с фреймворком Express или NestJS (предпочтительно для структуры).
- **База данных:** PostgreSQL (основные данные) + возможно отдельное хранилище для файлов.
- **Интеграции:** REST API, подключение LLM (ИИ-тестирование), хранение 3D-моделей (ссылки на облако/CDN).
- **Автоматизация:** n8n для обработки заявок на экскурсии/отправки уведомлений предприятиям.

**Задача:** Опиши полный план работ в виде текстового списка задач, разделенного на этапы. Код писать не нужно, только суть действий и требования к реализации. Учитывай, что в MVP не входят: мобильное приложение, чат, автоматическая генерация 3D, интеграция с 1С.

**Структура ответа (Этапы работ):**

### Этап 1: Инициация и настройка инфраструктуры
1. Инициализация репозиториев: создание структуры монорепозитория или отдельных папок `client` (Vue) и `server` (Node.js).
2. Настройка сервера Node.js: выбор фреймворка, установка зависимостей для работы с БД, CORS, переменные окружения.
3. Настройка клиентской части: создание проекта Vue 3 с использованием Vite, установка Pinia и Vue Router.
4. Проектирование схемы базы данных PostgreSQL. Определение таблиц: пользователи, роли, предприятия, вакансии, 3D-модели (ссылки), тесты, результаты тестов, записи на экскурсии, цифровой паспорт соискателя.
5. Подготовка Docker-окружения (опционально, но желательно) для локальной разработки с PostgreSQL.

### Этап 2: Разработка модуля аутентификации и ролевой модели
1. Реализация регистрации пользователя по полям: имя, email, роль (соискатель/студент). Пароль в MVP не указан, но реализовать вход по ссылке или временному коду, отправляемому на email, либо стандартный пароль.
2. Разработка middleware для проверки ролей и защиты API-маршрутов.
3. Создание структуры профиля для сохранения "Цифрового паспорта" (связь пользователя с его результатами тестов и предпочтениями).

### Этап 3: Разработка Административной панели (Админка для команды)
1. Создание раздела в Vue для команды проекта (без сложной регистрации, отдельный entrypoint или флаг доступа).
2. Реализация CRUD операций для сущности "Предприятие": название, описание, ссылка на логотип, контакты.
3. Реализация CRUD операций для "Вакансий": привязка к предприятию, текст должностной инструкции, порядок расчета ЗП, коллективный договор (PDF или текст).
4. Реализация поля для загрузки 3D-моделей: валидация типа файла или сохранение внешней ссылки на облачное хранилище.
5. Функция просмотра списка заявок на экскурсии и их статусов.

### Этап 4: Разработка ИИ-модуля тестирования и подбора вакансий
1. Проектирование структуры теста: создание таблиц с вопросами и вариантами ответов. Вопросы: интересы, готовность к переезду, ограничения по здоровью, ожидания по ЗП.
2. Разработка серверного endpoint для получения вопросов и отправки ответов.
3. **Интеграция с LLM (Large Language Model):**
   - Создание сервиса-абстракции для смены провайдера (OpenAI совместимый, OpenRouter).
   - Формирование промпта для LLM: передача ответов пользователя + JSON всех вакансий из БД.
   - Парсинг ответа LLM для получения списка ID наиболее подходящих вакансий.
   - **Важно:** Реализовать fallback-алгоритм (если LLM недоступна, подбор идет по жестким фильтрам: город = город предприятия или метки навыков).
4. Сохранение результатов теста и подобранного списка вакансий в "Цифровой паспорт" пользователя. Опиши в админке crud Цифрового паспорта.

### Этап 5: Разработка клиентского интерфейса для Соискателя/Студента
1. Верстка современного лендинга с основной информацией о проекте. Продумай лаконичный строгий дизайн, светлые цвета с контрастным шрифтом, подходящие по смыслу изображения.
2. Реализация компонента прохождения теста (step-by-step форма на 5-10 вопросов) - оформи в дизайне сайта.
3. Страница "Мои рекомендации": вывод карточек предприятий и вакансий, полученных от ИИ. Визуализация сходства/соответствия. оформи в дизайне сайта
4. **Визуализация рабочего места:** Интеграция плеера для просмотра 3D-моделей (на базе Three.js или встраивание iframe с облачного сервиса, например, Sketchfab Viewer).
5. Страница конкретной вакансии: отображение вкладок с должностной инструкцией, порядком расчета ЗП, коллективным договором.
6. Форма записи на экскурсию: выбор даты/времени (офлайн или ссылка на онлайн-трансляцию), кнопка "Записаться".
7. предусмотри в структуре главного меню сайта переходы по указанным разделам

### Этап 6: Автоматизация процессов с помощью n8n
1. Развертывание n8n (локально или в облаке).
2. Создание workflow для события "Новая запись на экскурсию":
   - Триггер: Webhook от бэкенда при создании заявки.
   - Действие: Поиск email представителя предприятия в БД.
   - Действие: Отправка письма на email предприятия с данными соискателя и деталями экскурсии.
3. Создание workflow для события "Отклик на вакансию" (аналогично уведомление).

### Этап 7: Реализация функционала экскурсий (MVP версия)
1. Разработка эндпоинта для скачивания/отправки направления на экскурсию (генерация простого PDF-файла с данными предприятия и соискателя).
2. Страница "Мои записи" в личном кабинете пользователя, где отображается статус заявки и кнопка "Скачать направление на экскурсию".

### Этап 8: Хранение данных и безопасность
1. Настройка облачного хранилища (S3-совместимого) для статики (логотипы, PDF договоров) и тяжелых файлов 3D-моделей.
2. Настройка валидации входных данных на бэкенде (защита от XSS и SQL-инъекций через ORM).
3. Реализация логирования действий пользователей.

### Этап 9: Тестирование по критериям успешности из ТЗ
1. Проверка сценария регистрации и прохождения теста.
2. Проверка функционала администратора: добавление минимум 5 предприятий с вакансиями.
3. Проверка записи на экскурсию и отображения заявки у администратора.
4. Нагрузочное тестирование страницы с 3D-моделью.

### Этап 10: Деплой и передача проекта
1. Подготовка инструкции по развертыванию (README.md).
2. Настройка CI/CD (например, GitHub Actions) для сборки Vue-приложения и деплоя Node.js сервера.
3. Заполнение базы данных реальными материалами от команды проекта (тексты описаний предприятий).
